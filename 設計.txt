----------------------------------------------------------------
必要そうな演出とかページ数とか

①ヘッダーがページスクロールについてくるようにしたい。

②複数ページの実装練習とか見やすくするためとかの理由で
　自己紹介、実績紹介、コンタクトフォームは全て別のHTMLを用意したい。
　（cssもどうせ見づらくなるだろうから複数作るつもりで考えておく）

③フッターは普通でいいような気がする。

④レスポンシブは対応する。(must)
　対応しているという事実がアピールになりそう。頑張る。

⑤画像編集は最低限にする。
　どうせデザインは下手くそなんだから良さそうなページを真似する方向で。
　背景画像はなんか海外のフリー背景素材とかを利用する。
　チャンネルのロゴみたいなのは考えておくと楽かもしれない。

⑥せっかくだから実績紹介ページはスライドショーにしてみたい。
　jsの教本にやり方描いてた気がする。

⑦node.jsも導入したしなんかパッケージ使って作成してみる。
　jsframeがめちゃめちゃよかったのでワクワクしてる。
　jQueryもちゃんと勉強する。

⑧スタイルシートはSassを使う。めちゃ便利だった。
　プロゲートではscssだったので書き方はscssにする。

コンセプト

・なるべくシンプルに。使う色は３〜４色くらい
    背景の色は白〜薄い色　もしくはugonoから雨イメージで青系
    装飾色は　①背景にした色を濃くしたもの　②逆相の色　③黒か白

・トップページ、自己紹介ページ、実績ページで最低３種類作る。
※トップページと自己紹介ページは一緒にしてもいいかもしれない。

・自己紹介は
    顔写真or似顔絵orロゴ
    簡単な自己紹介（p要素で片付くようなレベル）
    職務履歴とか学歴とか
の３つの要素を含める。

・実績ページは
    最上部に今までの実績のサムネイルをスライドショー形式にする。
    下部に実績のサムネイルとページの簡単な説明、実際のページに遷移するリンクをおく。
    
    もしくはworkフォルダを作って１作品１HTML用意する？追加するだけならそっちの方が管理も楽そう。（大枠をコピペできる）
    問題はファイルがどんどん増えていくこと。

・トップページは
    自己紹介から顔写真と簡単な自己紹介だけ持ってくる
    実績ページからスライドショーだけ持ってくる

必要になりそうなファイルとかclassとかidとか（順次追加、それぞれの簡単な解説を必ずつける。）

トップページ{
    index.html  トップページ{
        body 
            "container"クラス       bodyタグ直下に用意するコンテンツ全体のサイズ管理用クラス
            "aboutMe"クラス/id      自己紹介用顔写真とかを収納するクラス
            "myHistory"クラス       職務経歴・学歴などを収納するクラス
            "workSlideShow"クラス   実績をスライドショーにするためのクラス 実績増えたら簡単に追加できるようにしたい
            "workThumb"クラス       実績のサムネイルを示すクラス。imgタグにつける？workSlideShowに収納する？
    }
    main.scss    トップページのcss
    main.js     トップページのjs
}

ヘッダーとフッター{
    common.html ヘッダーとフッター{
        body{
            header{
                "headerContainer"クラス ヘッダー全体を収納するクラス　headerタグ自体にスタイル適用できればいらんかも
                "headerLogo"クラス      ヘッダーのロゴを収納するクラス　左寄せする
                "headerNav"クラス       ヘッダーのナビゲーション要素を収納するクラス　右寄せする
                "btn"クラス{
                    "btnAbout"id        自己紹介へ移動するスクリプト用
                    "btnWork"id         実績へ移動するスクリプト用
                    "btnHistory"id      経歴・学歴に移動するスクリプト用
                }   ボタン実装した場合に追加するクラス　いらなさそうなら省く　なんならaで良さそう
                "contact"クラス         コンタクトフォームへ遷移するクラス
            }

            footer{
                "footerContainer"クラス フッター全体を収納するクラス　ヘッダーと同様
                "footerLogo"クラス      フッターのロゴを収納するクラス　左寄せ　ヘッダーのと分ける必要がなさそうなら統一する。
                "footerNav"クラス       フッターのナビゲーション要素を収納するクラス
            }
        }
    }
    common.scss  ヘッダーとフッターのcss
    button.js   ボタン要素を採用した場合のjs
}
